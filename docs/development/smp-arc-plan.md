# 1.46.x SMP arc — sub-bite decomposition + the review's must-fix corrections

**Goal:** let APs (cores 1–3) run the scheduler concurrently with the BSP — flip `smp_wake_enabled=1` (main.cyr) and have each AP arm its LAPIC timer + run `sched_next`/`do_context_switch` under PER-CPU state with REAL locking, so `smp-smoke` shows `cpus online: 4` and the full battery stays green with 4 cores live.

**Source:** a 9-agent design+adversarial-review workflow (2026-06-24). The decomposition spine is sound (all 3 reviewers); the corrections below are mandatory and fold into the sub-bites they name.

**Safety invariant:** sub-bites 1–5 land while APs stay PARKED (`ap_entry` still `hlt`s), so single-core smokes stay green throughout. The wake flip is LAST and iron-gated. `smp_wake_enabled=0` fully reverts to the single-core MVP at any point.

## Sub-bites (ordered; never flip the wake until 1–6 are in + single-core-green)

1. **Restore the xchg spinlock primitive.** ✅ **DONE 2026-06-24** — `pmm_spin_lock`/`pmm_spin_unlock` (pmm.cyr) restored to the iron-proven xchg test-and-set (structured identically to smp.cyr `spin_lock`, no param → frame offsets verified byte-identical). FULLY single-core-validatable (lock always wins uncontended); `check.sh` 11/11, agnsh-smoke PASS. The `+58%` pmm cost is re-incurred deliberately as the foundation.
2. **Per-CPU scheduler identity** — `proc_current` / `preempt_count` / `sched_idle_pid` / `kpti_kernel_cr3`/`kpti_user_cr3` → APIC-id-indexed arrays (`tss_get_current_cpu_id()`, the `tss_array` template). On 1 core every array degenerates to `[0]` = the old global → fully single-core-validatable. **RENAME the old globals away so any missed read fails to COMPILE** (forcing-function).
3. **Lock the shared global structures** — `proc_table`/`proc_alloc_slot`, heap free-lists, `vfs_table`/write-fd pool, `sched_next`+state transitions. Replace every `preempt_disable`-as-exclusion (it's a SAME-CPU reentrancy gate, ZERO cross-CPU exclusion) with a real spinlock.
4. **Per-CPU SYSCALL stub** (HARDEST) — per-CPU syscall kstack + SYSRET cells + kpti CR3 pair, computed from the live APIC id in the regenerated stub. Carries the **r10 reload hazard**.
5. **Lock FS scratch + device rings + console** — the breadth worklist (ext2/fatfs buffers, NVMe/AHCI/r8169/virtio rings, fb console, kb_buf).
6. **AP runs the scheduler** — rewrite `ap_entry` to arm its LAPIC timer + mask LVT + sti + idle-yield loop, but keep `smp_wake_enabled=0` so it's INERT (compiles, single-core trivially green).
7. **THE FLIP** — `smp_wake_enabled=1` + let `sched_next` hand real procs to APs. CANNOT be TCG-proven; iron-gated, with a `smp_sched_aps` kill-switch so STEP 1 (wake + AP-idle-only) burns separately from STEP 2 (real procs on APs).

## MANDATORY corrections from the adversarial review (apply in the named sub-bite)

- **[sub-bite 1] Lock order is heap < pmm, NOT pmm < heap.** `kmalloc` takes `heap_lock` then calls `pmm_alloc` (`pmm_lock`) — pmm is the LEAF, acquired innermost → HIGHEST in an acquire-low→high order. Re-derive the whole chain from actual call graphs. (Noted in pmm.cyr.)
- **[sub-bite 1/3] No `spin_lock_at(addr)` with a parameter** until verified — the asm reads hardcoded `[rbp-0x10]`/`[rbp-0x08]`; a spilled param shifts them. Use fixed-global lock fns (copies of the proven structure) OR objdump-verify the generated frame offsets before trusting a parameterized form.
- **[sub-bite 2] `proc_current` is ~69 sites, not ~30** — and includes NON-stub C readers the plan missed: **`ring3.cyr:114` reads `kpti_user_cr3` directly** and `ring3.cyr:62/109/110/139` read `proc_current`. The rename-to-compile-fail forcing-function catches these.
- **[sub-bite 2] APIC-id `>=4` clamp** — the hand-emitted stub `and eax,0xFF` has no `>=4 → 0` guard; a non-contiguous APIC topology (Zen multi-die/CCX) overruns the `[4]` arrays. Either assert contiguous 0..3 ids at `smp_start_aps` and refuse `id>=4`, or emit a clamp.
- **[sub-bite 4] The r10 reload IS mandatory** — keep the per-CPU stack-load strictly in rax/r9 (copy smp.cyr:172-188 register discipline), NEVER r10, AND reload `r10 = &kernel_rsp_save_pc + cpu*8` AFTER the stack switch before the consuming `mov rdi,[r10]`. **Read the APIC id ONCE at entry into a provably-dead scratch** (NOT rax=syscall#, NOT r9=a4, NOT r10) — do not re-read 0xFEE00020 7× per syscall, and do not reuse rax (it's the syscall number, shuffled to rsi). **Disassemble the generated `syscall_entry_buf` (objdump -b binary) before trusting it.**
- **[sub-bite 4] Stop the execwait#37 runtime stub-rebuild** — rewriting the shared `syscall_entry_buf` tears the instruction stream if another CPU executes it. Make the stub LOAD a per-CPU current-syscall-stack pointer (per-CPU primary+nested pair); make `ew37_busy` per-CPU or under `sched_lock`.
- **[sub-bite 4] Split `syscall_init`** into `syscall_msr_init` (per-CPU: STAR/LSTAR/SFMASK/EFER/IBRS — APs run this) and `syscall_stub_build` (BSP-ONCE: emits the shared buffer). APs must NOT re-emit the shared buffer.
- **[sub-bite 5/6] `timer_handler` MUST branch on `tss_get_current_cpu_id()`** — there is ONE handler on vector 32; today it unconditionally does `timer_ticks++` + `net_rx_drain()` + `do_context_switch`. Only CPU0 may advance `timer_ticks` + drain the NIC; APs run ONLY `do_context_switch` + `apic_eoi`. This is a CODE branch, not prose — it is the #1 TCG-invisible corruption (double-counted timebase + reentrant `net_rx_drain` corrupting `net_rx_pkt`). `net_poll_busy` stays the same-CPU flag (don't pretend it's a try-lock).
- **[sub-bite 4/6] Stack-layout collisions in region 1** — AP1's trampoline stack `0x310000-0x320000` abuts CPU1 TSS.RSP0 top `0x320000` (first interrupt grows into the trampoline). Move AP RSP0 stacks off the trampoline addresses (or retire trampoline stacks after check-in); carve per-CPU syscall kstacks from a clean pool (the bite-1 region-7 neighborhood) with a guard, NOT 0x360000-style addresses that may overlap.
- **[sub-bite 3/7] `sched_lock` must be held across the WHOLE pick+state-flip+save/restore** — and add a per-proc `on_cpu` field: `sched_next` skips any proc with `on_cpu != -1` (running/mid-switch elsewhere). Releasing `sched_lock` before the context restore lets two CPUs run the SAME proc.
- **[sub-bite 6/7] Gate each AP's scheduling on a PER-CPU ready flag** the AP sets AFTER creating its idle proc + initializing `proc_current_pc[my]`, NOT on the global `sched_active` (which the BSP flips at a moment unrelated to AP readiness).
- **[sub-bite 6] LVT mask is the LEADING-but-UNCONFIRMED keyboard-death fix** — mask AP LINT0 (0xFEE00350)/LINT1 (0xFEE00360). It's iron-only/QEMU-invisible; if it doesn't fix the regression, the flip regresses keyboard on iron → the `smp_sched_aps` kill-switch + STEP-1-alone burn is the mitigation. Also calibrate the AP LAPIC timer count per-CPU (don't blind-copy the BSP's QEMU-tuned count — real-Zen LAPIC bus differs).

## Validation reality

TCG smokes CANNOT catch SMP races (scheduler selftests are TCG-only; KVM races kybernet at boot). So single-core green is NECESSARY but NOT SUFFICIENT for sub-bites 4/6/7 — those need careful-KVM (`-smp 4` + 2 procs spinning getpid/write, long dwell) + iron on archaemenid (the ONLY place keyboard-death + real-Zen INIT-SIPI timing manifest). Burn order via `smp_sched_aps`: STEP 1 (wake + AP-idle-only) first, THEN STEP 2 (real procs on APs). Read the framebuffer photo for iron diagnosis, never CMOS.
